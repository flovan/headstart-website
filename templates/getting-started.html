Setting up Headstart for your next project is easy as can be. If you're not familiar with a Command Line Interface (CLI), don't fear—just follow this easy guide and enjoy the ride!

Get the files

To get started with your project, you need a fresh copy of the source files. First you need to decide if you want the Main or the Lite version of Headstart.

Main version

Includes page-specific scripts and styles files, with a more advanced JS setup.

Download .zip or Fork Headstart at Github

Lite version

Includes one main scripts and styles file, with a basic JS setup.

Download .zip or Fork Headstart Lite at Github

First-time setup

If this isn't the first time you're using Headstart on your machine, you can skip to the next step!

Since this framework is built on open-source software, you are required to install a few things before you can get started. A small trade-off for not having to pay a dime, right?

It might seem like a lot of work at first, but remember, you' be saving time later on, and this setup only needs to be done once!

Command Line Interface (CLI)

You'll be using a CLI quite a lot since there is no GUI to use Headstart. Depending on your Operating System, you will probably end up with one of following:

  • OS X: Terminal.app ~ Applications/Utilities
  • Windows: Command Prompt ~ Start > All Programs > Accessories > Command Prompt
  • Linux: Shell Prompt ~ Hell, you're using Linux, you tell me where it's at!

So open that bad boy up and onwards we go!

Installing Ruby

Untill the C/C++ port of the Sass compiler is fully featured, we have to rely on the original Ruby implementation to take care of the Scss compilation.

OS X usually comes with Ruby installed, but you make sure by running ruby -v in your CLI. If the result is any version of Ruby, you can continue to the next step!

If you are presented with command not found, you can follow instructions on this page to set it up.

Installing Sass

After confirming Ruby is installed, the next step is installing Sass. This is rather straightforward, and can be done by running gem install sass

Installing NodeJS and NPM

NodeJS is what ties together the modules that make up Headstart. NPM (Node Packaged Modules) is the package manager that comes with Node.

You can download the appropriate installer through the website.

Installing Gulp

Next up is Gulp. Gulp is a streaming build system, that makes it easy to hook up files to modules and back into files. It is a Node module itself, so installing it is a piece of cake.

Run sudo npm install -g gulp to install Gulp globally (that's what the -g is for).

Running the commando in administrator mode by using sudo will suppress possible errors. Don't forget to confirm with your user password when asked.

That's all folks!

Congrats on getting through that lengthy setup. Never again! Unless you'll be working on a different machine of course ;)

Every-time setup

What? Another setup? Every time?

Fear not, brave front-end developer, the routinely setup requires nothing but a quick NPM command.
Run sudo npm install and watch the modules roll in.

That's it? Yes! Just open up the files in your favourite editor and start up the engine.

Configuration

There's room for a little configuration throughout the framework, though usually the only bits you'd have to change on a per-project basis will be HTML or CSS related.

Global settings

Open up gulpfile.js in the root directory. This file contains the code that does all the heavy lifting for you. It is commented throughout, though the only part you should bother touching would be the config, just below the required modules, at the top.

Structure

Headstart makes file management pretty dang easy, leaving you with a clean folder structure to get you thinking in fragments. Did I include that js library in the HTML yet? Is the file present in your libs folder? Then that' all you have to do.

Time to get acquainted some of the root files, and of course the assets folder!

Root files

The root of the project contains a bunch of files, of which the following should be of your interest:

  • gulpfile.js ~ The code file that does the heavy lifting for you.
  • .htmlhintrc ~ Validation rules for the HTML.
  • .jshintrc ~ Validation rules for the JS.

HTML

The first subfolder of the app directory is where the HTML lives. Read all about what is packaged by default in the HTML chapter.

Images

The images folder is empty by default, with the exception of the icons subfolder. In here you will see placeholder icon images that you can override with your own assets. They are only provided so you can easily remember what sizes you need.

Don't forget to remove the comments that surround the link and meta tags in the HTML. Otherwise updating the images with your assets will not yield any results.

Javascript

Read all about what is and isn't provided out of the box in the JS chapter.

Miscellaneous

Headstart comes with some default files (of which some will only be added at production runtime):

  • crossdomain.xml ~ By default other domains are not allowed to access any (nested) files that are hosted next to this file. A less restrictive policy is provided in comments as well.
  • favicon.ico ~ Not very straight forward to have an icon in this location, but IE doesn't understand PNG favicons. This is—unfortunately—the only way to support all IE versions.
  • htaccess.txt ~ Regular .htaccess files are hidden files by default. During development and in production mode, this file is transformed into its working state. By default it adds support for webfont filetypes, and enables gzipping for HTML files and assets. There is also a part that will enable caching of files, though it is commented out for obvious reasons.
  • humans.txt ~ We are people, not machines! Read more on this here.
  • robots.txt ~ Tell robots if they can crawl through your content or not. Read more here.

Sass

If you aren't using Sass or Scss yet, you are really missing out! Read up on it if you haven't already! Read all about what Headstart offers (and what not!) in the CSS chapter.

Usage

Can you feel the excitement in the air?

Starting Headstart only takes one command. Remember Gulp, the streaming build system?
Kickstart the engine by running the following command in your CLI:

gulp

Now watch as each task meticulously passes through the files. Then—a page opens. And you're off!
Start adding in the HTML structure and watch the page refresh on each save. Add in your CSS and Javascript and wonder how you have ever managed to program without it.
