Planned changes

  • Make things faster!
  • Use the C/C++ port of Sass when it has matured enough.
  • Use a cache manifest and revision filenames.
  • Make it possible to validate HTML through W3C.

Known bugs

  • When using <code> blocks, some characters (such as <) will crash the building process.
  • Adding a new folder into any watched directory (such as sass or js) crashes the building process.
  • Newly added files sometimes escape the watch, thus requiring you to quit and restart the process.
  • When importing a scss module into your main stylesheet, livereloading won't occur through saves in the imported module until the Headstart build is restarted. Workaround: save the main stylesheet where you imported the module.

Any bugs can be added through the Github issues page.
