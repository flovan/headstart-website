The default boilerplate files comes bundled with a few libraries and patches, and a basic setup to get you started right away.

Folder structure

  • ./ie ~ Patches for IE<9.
  • ./libs ~ Plugins and third-party libraries.
  • ./ ~ Main and page-specific code.

Note: Files prefixed with an underscore will be excluded from the build process.

IE files

This folder contains patches and polyfills for IE<9, ordered in the body and head folders. Each folder will produce a concatenated file which you can use through the conditional comments in the layout templates:

templates/layout/header.html:

<!--[if lt IE 9]>-->
<script src="assets/js/ie.head.min.js"></script>
<!--<![endif]-->

templates/layout/footer.html:

<!--[if lt IE 9]>-->
<script src="assets/js/ie.body.min.js"></script>
<!--<![endif]-->

Library files

$

Headstart comes with a Jeesh build of Ender to replace the much heavier jQuery 1.11.x, which is also bundled, but excluded from the build. Feel free to revert back to jQuery if you want. Remember that you can use a custom fork of the boilerplate files to manage your own defaults. The Jeesh comes with the following libraries and replaces almost every commonly used aspect of jQuery, but at half the weight.:

  • domReady ~ A cross-browser domReady.
  • Qwery ~ A fast light-weight selector engine.
  • Bonzo ~ A bullet-proof DOM utility.
  • Bean ~ A multi-platform event manager
  • Underscore ~ A functional programming toolbelt.

Also bundled, but excluded from the build, is an extended version of this Jeesh build, which contains extra libraries:

  • Reqwest ~ Browser asynchronous http requests aka AJAX.
  • Morpheus ~ A brilliant animation library.

Prefix the unused libraries with an underscore to prevent abuse of the $.

Utils

A globally-accessible wrapper for your utilities. By default the class can:

  • Check if the client device is touch-enabled is available.
  • Detect oldy (IE 9-) browsers.

Loose libs

  • Fastclick ~ Getting rid of touch-delays.
  • Underscore ~ A functional programming toolbelt. Excluded by default.

Dev libs

  • Log ~ Cross-browser logging with both log() and console.log().

Note: When running --production:

  • Dev libraries will get excluded when building
  • All log() and console.log() statements will get stripped automatically.

Root files

Main script

The main.js file contains a very basic .ready() setup wrapped in an IIFE.

View specific scripts

View-specific files make it easy to produce script files for individual pages. They are content-wise the same as the main.js. There is an example file included for the index page. To use it, remove the leading underscore from its filename.
