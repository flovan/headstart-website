Headstart provides you with a solid HTML foundation, based on the HTML5 Boilerplate, but extended with those things you always end up needing.

The default HTML is pretty light-weight and well commented. Nonetheless, here is a top-to-bottom overview to get you acquainted.

Note: If you are working with the Main version of Headstart, the names of your HTML files will each be tied to a "view". The build script uses those names to inject the correct assets.

First off, we set the document character set to be utf-8. Then we tell IE browsers to render as the latest version available, or to use the Google Chrome Frame (when available). You can read more on that here.

<meta charset="utf-8">
<meta http-equiv="X-UA-Compatible" content="IE=edge,chrome=1">

Next up is the SEO part. Filling this in is a small effort on your part, but it makes sure bots and spiders know what your page is all about.

<title>__PAGE_TITLE__</title>
<meta name="description" content="__PAGE_DESCRIPTION__">
<meta name="keywords" content="__PAGE_KEYWORDS__">
<meta name="author" content="__PAGE_AUTHOR__">

In the following bit, we tell bots and spiders to use our custom SEO meta tags, rather then the content they find on the page. We also tell Google not to try and translate our website, as the toolbar they use is just plain silly.

<!-- By default the robots tag will also be set to "follow,index". -->
<!-- More on that on http://www.robotstxt.org/meta.html -->
<meta name="robots" content="noodp, noydir">
<meta name="google" content="notranslate">

Up next is the mobile and viewport part. The first tag will make sure viewports play nicely with your beatifully crafted responsive site. If you want your users to be able to scale your website by pinching, you can remove user-scalable=no, maximum-scale=1.0 and minimum-scale=1.0 from the content attribute. The other meta tags target a grey area of devices.

Read all about the viewport and mobile meta tags over here.

<meta name="viewport" content="width=device-width, initial-scale=1.0, user-scalable=no, maximum-scale=1.0, minimum-scale=1.0">
<meta name="HandheldFriendly" content="true">
<meta name="MobileOptimized" content="320">
<meta http-equiv="cleartype" content="on">

Ah, mobile web apps. Or how iOS adds a sh*tload of meta tags to customize how your website is displayed and handled. By default, a few of them have been included in comments, though this gist does a better job showing you what is possible.

<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-title" content="__PAGE_TITLE__">
<meta name="apple-mobile-web-app-status-bar-style" content="black">

The last bit of default code contains the icon tags. One icon to rule all touch icons, a favicon for the smarter browsers, and settings for the Windows Phone tile. Handy dandy, right? If you want to learn more about these kinds of icons, you should read this page.

When using a favicon, don't forget to update the .ico file in the meta folder! Otherwise IE browsers won't display the favicon at all.

<link rel="apple-touch-iconprecomposed" sizes="152x152" href="images/icons/apple-touch-icon-152x152.png">
<link rel="icon" href="images/icons/favicon.png">
<meta name="msapplication-TileImage" content="images/icons/wp-tile-icon-144x144.png">
<meta name="msapplication-TileColor" content="#FFFFFF">

As a final note, it is worth mentioning that all scripts and styles will be injected for you at runtime. So please don' remove the following comment markers:

<!DOCTYPE html>
<html>
<head>
...
<!-- inject:css -->
<!-- endinject -->
</head>  
<body>
...
<!-- inject:js -->
<!-- endinject -->
</body>
</html>
