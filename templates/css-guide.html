Cascading Style Sheets Guide

{{> partials/sidebar_menu}}

Headstart uses Sass to compile stylesheets, and forces next to no default styles into your workflow. With just a few easy to use mixins and functions you can make any kind of layout happen!

Folder structure

Great Sass comes with great responsibility. By separating the responsibility of your styles into named chunks, finding out where a certain style might be getting applied becomes a lot easier. This is the default setup that you'll find inside of the assets/sass folder:

  • ./base ~ Files that don't generate CSS on their own.
  • ./libs ~ Libraries.
  • ./mixin ~ CSS generating modules.
  • ./module ~ Chunks of styles that are separated because of their specific purposes.
  • ./vendor ~ All third-party CSS that comes with eg. plugins.
  • ./ ~ The root files combine the different chunks and contain other, loose styles.

Prefixing

With the help of Autoprefixer you will never have to write a single prefix again. Ever! Generator functions are also a thing of the past. Start writing real CSS3.

Base files

Files that don't generate CSS on their own.

_config_global.scss

The global config file imports all mixins and functions to make them available for usage, and contains top-level variables:

  • $base-font-size ~ Defaults to 16px, the de facto browser font size.
  • $base-line-height ~ Defaults to 1.5, a good starting point for readability.
  • $total-columns ~ Defaults to 12, which devides easily for 2-, 3- and 4-column layouts.
  • $default-gutter-width ~ Defaults to 5% and adds fluid spacing in between columns.
  • $default-column-breakpoint ~ Defaults to 650px and indicates which space will make columns start floating.
  • $max-container-width ~ The maximum width a container can be, defaults to 1140px.
  • $min-page-padding ~ When the available space is less wide than the $max-container-width, this is the amount of side-padding that will be added.

_config_site.scss

This file defines project specific variables, such as colors, and the default font-stack:

$font-stack: Helvetica Neue, Helvetica, Verdana, Roboto, sans-serif;

_functions.scss

Some functions that are also used by included mixins:

  • strip($number) ~ Will string the unit from a passed value.
  • em($target, $context: $base-font-size) ~ Generates em values from a number. Supports negatives as well. The $context variable is used as base value during calculations.
  • rem($target) ~ Generates rem values with px fallbacks from a number. Also supports negatives.
  • tint($color, $percent) ~ Will lighten a color with a percentage of white.
  • darken($color, $percent) ~ Will darken a color with a percentage of black.

_state.scss

A collection of silent classes that make it easy to keep your stylesheets DRY:

  • %is-hidden ~ Visually hide an element. Handy for image replacements.
  • %is-bold
  • %is-underlined
  • %is-italic
  • %is-uppercase
  • %is-smallcaps
  • %is-cf ~ Clears floats.
  • %is-vertical-aligned ~ Alignes stuff vertically in its parent.

To use these in your HTML, you can extend from them, for example:

.capslock { @extend %is-uppercase; }

It is still advised to use <strong></strong> for bold and <em></em> for italic.

Mixins

The default mixins provide core functionalites such as the grid-system, media queries, etc. The Gzipping—enabled by the .htaccess file—will cache and strip out strings with multiple occurences, so file size is never an issue.

_all.scss

Simply imports all the other mixins. It is imported itself into _config_global.scss.

_baseline.scss

A mixin to make baseline alignment easier to implement. Works great together with the Basehold.it stylesheet.

_grid.scss

The grid module allows you to easily make auto-collapsing columns:

section {
    /* Make the parent clearfix */
    @extend %is-cf;
}
/*
column(
    $columns, // How many columns to span
    $end: false, // Is it the end of a row
    $break: $default-column-breakpoint, // Where it collapses
    $context: $total-columns // The context
)
*/

aside {
    /* Make it float left and take up 4 out of 12 columns */
    @include column(4);
}
article {
    /* Make it close the row by floating right
       and taking in the remaining 8 columns */
    @include column(8, true);
}

A column starts being a column when the available width is at least the defined $break, which defaults to $default-column-breakpoint or 650px (as defined in the base files).

You can also have multiple column settings by wrapping the column includes in media queries (more on those in the next chapter). As long as the defined breakpoint is larger than the column breakpoint, the media-queries will delay it from floating. This is some example code that is used on the homepage:

div {
    /* Setup a 2 column layout up to 1049px */
    @include media-max(1049px)
    {
        /* Odd columns (1,3,5,...) will float left */
        &:nth-child(odd) { @include column(6); }
        /* Even columns (2,4,6,...) will float right */
        &:nth-child(even) { @include column(6,true); }
        /* First two blocks push down */
        &:nth-child(-n + 2) { @include rem(margin-bottom, 50); }
    }

    /* Set up a 4 column layout */
    /* Starting at 1050px */
    @include media-min(1050px)
    {
        /* All but the last column float left */
        &:not(:last-child) { @include column(3); }
        /* Last column floats right */
        &:last-child { @include column(3,true); }
    }
}

If you want to stray from the $default-column-breakpoint, you can use the $break parameter of the mixin to define a custom one:

/* The default value of 650px will be dropped in favour of 800px */
div { @include column(6, false, 800px); }

_media.scss

Headstart comes with 4 media query mixins that make it a painless process to tie styles to certain dimensions.

/* Un until given value */
@include media-max($value) {
    /* ... */
}

/* Starting from given value */
@include media-min($value) {
    /* ... */
}

/* Between given min and max */
@include media-min-max($valueMin, $valueMax) {
    /* ... */
}

/* On retina screens */
@include highres() {
    ...
}
  • Passed in values will be stripped of their unit and converted to ems.
  • All media-* functions have an optional $query parameter that defaults to 'screen'.

_rem.scss

Not to be confused with the likewise named function, this mixin outputs rem values with pixel fallbacks for a given property. Great for paddings and margins!

/* For example, using: */

div {
    @include rem(margin-top, 20);
    @include rem(padding, 10, 20, 10, 30);
}

/* Will result in */

div {
    margin-top: 20px;
    margin-top: 1.25rem;
    padding: 10px 20px 10px 30px;
    padding: 0.625rem 1.25rem 0.625rem 1.875rem;
}

_triangle.scss

CSS triangles, we all love them, but setting them up isn't really straightforward. This mixin makes it easier.

/* triangle($width: 10px, $height: 10px, $direction: up, $color: red) */

.tooltip:before {
    content: '';
    display: block;
    @include triangle(15px, 10px, down, #fff);
    /* ... */
}

Modules

Modules are grouped chunks of styles that are separated to lighten your main files.

_button.scss

Everything button goes in here. The placeholder class %button normalizes some default properties of the most commonly used button-elements, such as <a></a> or <button></button>. This is an example of how a stroked button class %button--stroked could be defined:

/* For usage in HTML, you can use a regular class,
   or extend a regular class from this placeholder */
%button--stroked {
    @extend %button;
    border: 1px solid #000;
}

/* Somewhere in your main styles */
.contact-submit-button {
    @extend %button--stroked;
}

_container.scss

Containers are centered blocks with a maximum width. When the available width is less than their defined maximum, they get padded on both sides. Images inside of these type of blocks are made fluid by default so they scale along beautifully. Extend your selectors from the placeholder as following:

/* This could be any class */
.header__container {
    @extend %container;
}

_font.scss

A place to put your @font-face configuration.

_form.scss

All form related styles go in here. Normalizes some default styles for forms elements.

_normalize.scss

This is simply the Normalize.css library.

_print.scss

The print module is limited to just a few styles that will improve the printing "experience" of your website, eg. making all the text black, removing background-images and shadows, etc. This file is imported into the main stylesheet to prevent an extra GET request.

_reporting.scss

This module makes it easy to add reporting paragraphs and lists.

Add a info-block with the class .report--info.

Add a warning-block with the class .report--warning.

Add a success-block with the class .report--success.

  • Lists are also supported.
  • Display errors with the .report--error class.

_text.scss

Everything text-related is contained in this module:

  • Text-selection color
  • Headings
  • Lists
  • Paragraphs
  • Links

Vendor files

The vendor folder is where you can put the stylesheets that sometimes come with plugins. SCSS stylesheets support regular CSS as well, so just change the .css extension to .scss and you're ready to start @importing them.

Root files

Main styles

This stylesheet gets included every page. The styles of, for example, a <header>, <menu> or <footer> are best defined here. If you plan on using modules, this is also the best place to import them.

View specific styles

When working on a large scale project, somes styles will be limited to individual pages. This is when the view-specific files come in handy, as they will lighten up the main stylesheet and thus speed up the individual pages of your website.

These files should be prefixed with view-, followed by the name of the target HTML file. For example view-index.scss, view-about.scss or view-portfolio.scss
